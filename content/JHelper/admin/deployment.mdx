# 빌드 및 배포


## 1. 빌드 및 배포

### 1.1. 프로젝트 빌드
프로덕션 환경에 배포할 정적 파일을 생성합니다.
```bash
yarn build
```
빌드 결과물은 `public/` 디렉터리에 생성됩니다.

### 1.2. GitHub Pages 배포를 위한 빌드
GitHub Pages와 같이 `docs` 디렉터리를 사용하는 환경을 위한 빌드 명령어입니다.
```bash
yarn build:ghpage
```
이 명령어는 `docs/` 디렉터리에 빌드 결과물을 생성합니다.

### 1.3. 배포 프로세스
- `.github/workflows/deploy.yml` 파일에 정의된 GitHub Actions 워크플로우를 통해 자동 배포됩니다.
- 일반적으로 `main` 브랜치에 푸시될 때마다 빌드 및 배포가 트리거됩니다.

### 1.4. `onPostBuild` 스크립트 (보안)
- `gatsby-node.js`에 정의된 `onPostBuild` 스크립트는 빌드된 파일에서 개발 환경의 절대 경로 정보(예: `C:\Users\oslab\develop\JHelper`)가 노출되는 것을 방지하기 위해 해당 경로를 제거합니다. 이는 프로덕션 환경의 보안을 강화하는 중요한 커스터마이징입니다.
