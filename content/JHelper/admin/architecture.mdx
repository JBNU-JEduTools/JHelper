# JHelper 기술 유지보수 가이드

이 문서는 JHelper 프로젝트의 기술적인 구조를 이해하고, 코드를 수정하며, 기능을 유지보수하는 데 필요한 모든 지식을 제공합니다.

---

## 1. 프로젝트 아키텍처

### 1.1. 프로젝트 핵심 구조

- **`gatsby-config.js`**:
    - 사이트의 전반적인 설정 (사이트 제목, 설명 등)
    - Gatsby 플러그인 설정 (파일 시스템 소스, MDX 처리 등)
    - **왼쪽 네비게이션 메뉴의 순서 및 구조 정의** (매우 중요)

- **`gatsby-node.js`**:
    - Gatsby 빌드 프로세스 중 노드 생성 및 페이지 생성 로직 정의
    - `onCreateNode`: MDX 파일로부터 URL 슬러그(slug) 생성
    - `createPages`: 생성된 슬러그를 기반으로 각 문서 페이지를 동적으로 생성
    - `onPostBuild`: 빌드 완료 후, 빌드 결과물에서 불필요한 절대 경로를 제거하는 **보안 스크립트** 포함

- **`content/` 디렉터리**:
    - 모든 문서 콘텐츠(MDX 파일)가 저장되는 최상위 디렉터리
    - 이 디렉터리 내의 폴더 구조가 웹사이트의 URL 경로 구조에 직접적으로 반영됩니다.

- **`src/templates/docs.js`**:
    - 모든 문서 페이지를 렌더링하는 데 사용되는 **단일 템플릿 파일**
    - GraphQL 쿼리를 사용하여 특정 페이지의 MDX 콘텐츠를 가져와 `MDXRenderer`로 렌더링

- **`src/components/Layout.js`**:
    - 웹사이트의 전체적인 레이아웃을 정의하는 React 컴포넌트
    - 헤더, 왼쪽 네비게이션 사이드바, 메인 콘텐츠 영역, 오른쪽 목차 사이드바 등을 포함

- **`package.json`**:
    - 프로젝트의 메타데이터, 스크립트 명령어, 개발 및 프로덕션 의존성 목록

### 1.2. 데이터 흐름

JHelper는 다음과 같은 흐름으로 MDX 파일을 웹 페이지로 변환합니다.

1.  **콘텐츠 작성**: 관리자가 `content/` 폴더 내에 `.mdx` 파일을 작성합니다.
2.  **파일 감지**: `gatsby-source-filesystem` 플러그인이 `content/` 폴더의 파일들을 감지하여 Gatsby의 데이터 레이어로 가져옵니다.
3.  **페이지 생성**: `gatsby-node.js`의 로직이 각 파일에 대한 고유한 URL(`slug`)을 생성하고, `docs.js` 템플릿을 사용하여 페이지를 동적으로 만듭니다.
4.  **데이터 요청**: 사용자가 페이지에 접속하면, `docs.js` 템플릿의 GraphQL 쿼리가 해당 페이지에 필요한 데이터(본문, 제목 등)를 요청합니다.
5.  **렌더링**: `MDXRenderer` 컴포넌트가 GraphQL로 받은 데이터를 최종 HTML로 렌더링하여 사용자에게 보여줍니다.

## 2. 주요 기술 스택

- **`gatsby`**: React 기반의 정적 사이트 생성 프레임워크
- **`react`, `react-dom`**: UI 컴포넌트 개발
- **`@mdx-js/react`, `gatsby-plugin-mdx`**: Markdown 문서 내에서 JSX(React 컴포넌트)를 사용할 수 있게 해주는 도구
- **`theme-ui`, `gatsby-plugin-theme-ui`**: UI 스타일링 및 디자인 테마 관리

---

## 3. 핵심 유지보수 작업



### 3.1. UI 및 스타일링 수정

프로젝트의 전반적인 디자인(색상, 폰트, 간격 등)은 `Theme UI`를 통해 관리됩니다.

- **핵심 설정 파일:** `src/gatsby-plugin-theme-ui/index.js`
- **수정 예시 (기본 색상 변경):** 위 파일의 `colors` 객체에서 `primary` 또는 `heading` 값을 수정하면 사이트 전체의 색상이 변경됩니다.

### 3.2. 빌드 및 배포 스크립트 이해

`package.json`의 `scripts` 섹션에는 프로젝트 관리를 위한 주요 명령어들이 정의되어 있습니다.

- `develop`: 로컬 개발 서버를 실행합니다.
- `build`: 프로덕션용 정적 파일을 `public/` 폴더에 생성합니다.
