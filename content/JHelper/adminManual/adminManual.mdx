# 관리자 추가

- 관리자의 변경이 필요한 경우(연구실 인원 변동 등)에 관리자의 추가가 필요함

  <img src="./Images/addadmin1.png" width="800px" height="auto" /> <br />
  → 상단 메뉴 중 “Settings" 클릭 <br />
  <img src="./Images/addadmin2.png" width="800px" height="auto" /> <br />
  → 좌측 메뉴 중 “Collaborators and Teams" 클릭 <br />
  <img src="./Images/addadmin3.png" width="600px" height="auto" /> <br />
  → 상단 메뉴 중 “Add People" 클릭하여 Github 닉네임 입력 후 Invite <br />
  <img src="./Images/addadmin4.png" width="600px" height="auto" /> <br />
  → Base Roll이 Admin이기 때문에 인원 추가 시 자동으로 관리자로 지정이 됨

# Pull Request

- Github Repository 내의 PR 요청을 확인하고 그 내용을 Review

  <img src="./Images/PR_1.png" width="600px" height="auto" /> <br />

- 이상이 없을 시 PR 요청을 Merge 하면 PR의 내용을 Action으로 넘겨서 Build & Test 수행
  <img src="./Images/PR_Merge.png" width="600px" height = "auto" /> <br />
 
# Merge 이후 Github Action을 이용한 Build & Test

- Merge 이후에는 Github Action을 통해 자동으로 Build & Test를 진행하도록 설정이 되어 있음

- Github Action을 확인하여 Merge 한 내용이 이상 없이 잘 작동하는지 확인이 필요

- 방법은 아래와 같음

  → 상단 메뉴 중 “Action" 클릭 <br />
  <img src="./Images/Action_Detail.png" width="600px" height="auto" /> <br />

      - 노란색 점으로 나타나 있는 부분이 현재 Test가 진행 중인 부분 

  → “Build"가 진행 중인 사항 확인 <br />

  <img src="./Images/Action_1.png" width="600px" height="auto" /> <br />

        - 현재 진행 중인 상황 확인 가능 
        - build 클릭 시 상세 진행 사항 확인 가능

  → 진행 중인 상황에서 build 클릭하여 상세 진행 사항 확인 시 아래에 같이 나오게 됨 <br />
    <img src="./Images/GH_Action.png" width="600px" height="auto" /> <br />
